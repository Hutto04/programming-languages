Project Description: Harmony Community Center Website

Objective:

Create a fully functional, multi-page website for Harmony Community Center, a fictional local hub offering various classes, events, and facilities to the community. The website should provide information about the center, allow users to sign up for newsletters, register for classes, and contact the center through a form.

Features and Pages:

  1. Homepage:
    • Introduction to the community center with a welcoming message, news highlights, and upcoming events.
    • Use semantic HTML to structure content and embed a video tour of the center.
  2. About Us Page:
    • Details the history, mission, and values of the center.
    • Include images of the facilities and staff bios using <figure> and <figcaption> for captions.
  3. Classes and Events Page:
    • Lists available classes and events.
    • Use HTML tables for schedules and custom data attributes to store additional information like class capacity or event age restrictions.
  4. Registration Form:
    • Allows users to register for classes and events.
    • Include various input types (text, email, date, dropdowns for class selection, etc.) and demonstrate form validation using HTML5 attributes.
  5. Contact Us Page:
    • Contains a contact form for inquiries.
    • Include fields for name, email, subject, and message.
    • Showcase the use of the <textarea> for multi-line input and include a Google Maps embed of the center's location using the <iframe> tag.
  6. Membership Sign-up Page:
    • A simple sign-up form for newsletters and membership updates.
    • Emphasize the use of the email input type and checkboxes for subscription options.
  7. Gallery Page:
    • A photo gallery of past events and classes.
    • Utilize the <figure> and <img> tags and implement a lightbox feature using HTML5's data attributes and JavaScript.

Technical Requirements:

  • Semantic HTML: Use semantic tags effectively to structure content for accessibility and SEO.
  • Forms and Input: Implement forms with validation for user registrations and contact inquiries.
  • Multimedia Embedding: Embed videos and images that enhance the user experience.
  • HTML5 APIs: Where applicable, introduce advanced features like the Canvas API for an interactive event calendar (a simplified example).
  • Responsive Design Considerations: Although primarily an HTML project, mention the use of responsive design techniques (e.g., using the <meta> viewport tag) to ensure the website is mobile-friendly.

Deliverables:

  • HTML Source Code: Well-organized and commented HTML files for each page, including external links to CSS for styling (if used).
  • Documentation: A brief document explaining the structure of the website, the choice of HTML elements, and any challenges encountered during development.
